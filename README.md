## Верхнеуровневое описание проекта

Веб-приложение, позволяющее:

* задать **силу команд** (в терминах матбоев - результаты командной олимпиады).
* **составить турнирную сетку**.
* **смоделировать проведение турнира** множество раз.
* визуализировать распределение возможных итоговых мест и исходов матчей.
* транслировать результаты текущего турнира.
___

## Цели проекта
__
  1. Первоначальной целью ставится удовлетворение запроса заказчика: возможность прямой трансляции результатов турнира, а также удобство пользования приложением для администраторов.
  2. Предоставить свободному пользователю возможность смоделировать собственные турниры, их результаты при различных вводных данных.
  3. Исследовать влияние командной (распределительной) олимпиады, турнирной сетки и количества лиг (групп) на результаты всего турнира математических боев.
___
## Предварительное исследование

Необходимо

  1. Выгрузить имеющиеся данные по результатам турниров.
  2. Выявить зависимость вероятности победы от изначального распределения сил.

Для исследования будут использованы ресурсы нейронных сетей в целях парсинга данных. Библиотека Pandas для визуализации данных. Предварительная гипотеза: **можно использовать методику, которая применяется в шахматах**

## Возможности создаваемого приложения.
Приложение представляет собой 4 вкладки.
  1. Вкладка администратора.
  2. Вкладка-конструктор.
  3. Вкладка для трансляции текущего турнира.
  4. Вкладка с общей информацией о самом турнире (свод правил, ЧаВО).

Внизу каждой страницы основная информация об организации (наименование ЮЛ, контактные данные)
Возможности каждой вкладки описаны ниже
### Вкладка администратора

Это вкладка, на которой виден список всех администраторов сайта. При ее открытии требует залогиниться/зарегистрироваться, если пользователь зашел без аккаунта.
На вкладке можно:
  1. Добавить нового администратора по логину (только если такой логин существует).
  2. Удалить добавленного ранее администратора (или любого администратора, который находится строго ниже по иерархии. Здесь "строго ниже" означает, что права были выданы администратором, который получил права от пользователя, или же ниже по цепочке вниз).
  3. Аналогичные действия с ролью "редактор" - удаление и добавление. Редактор - более низкая роль, ее пользователя может лишить любой администратор вне зависимости от того, кто роль выдал.

### Вкладка-конструктор

Здесь можно смоделировать свой турнир. Вводные можно задать тремя способами:
  1. Занести собственный csv в определенном формате.
  2. Самостоятельно завести данные (задать количество команд, силы каждой).
  3. Воспользоваться встроенным рандомайзером (вручную заведя количество команд и далее по нажатию кнопки случайным образом раздать "силы").
После можно выбрать
  1. Количество групп (и задать количество команд в каждой группе). Не более 100, дабы не перегружать мощности. Если симуляций 100, то $n^2 \cdot 100$ - примерная асимптотика прогона 100 турниров. $10^6$ операций - логичная верхняя грань. 
  2. Турнирную систему (швейцарская, круговой турнир, на вылет (новые могут добавиться в будущем)). По умолчанию швейцарская, как на самом турнире.
  3. Собственную формулу вероятности победы команды в зависимости от соотношения сил (функция от x, y в формате Wolfrahm. По умолчанию будет написана полученная при исследовании).
Далее запускается симуляция из 100 турниров. Выводится таблица: для каждой команды ее название, лига и среднее место по итогам 100 турниров.

### Вкладка для трансляции текущего турнира.

У вкладки в оглавлении название и номер турнира большим шрифтом. Под ними мелким город проведения (Казань) и год проведения. При открытии от имени администратора в панель можно добавлять новые классы (также можно удалять классы, если на турнире изменился список). На каждой панели класса есть
  1. Общая сводка по каждой лиге (Квадратная таблица с результатами. Пример на картинке)
